# API / System Design / RAG / Transformer Study Notes

This repo contains a single-page, glance-friendly technical interview study guide.

Open `index.html` in a browser to review.

## Focus Areas
- API design principles (contract, idempotency, auth, retries, errors, versioning, observability)
- System design principles (FR vs NFR, PASS CURES, capacity estimation, tradeoffs)
- RAG and agentic retrieval/verification loop (including metrics placement)
- Transformer architecture (QKV, self-attention, decoder-only/GPT-style framing)

## Files
- `index.html` - The main study notes page
- `styles.css` - Theme and layout
- `script.js` - Search, expand/collapse, dense mode

## How To Read It
- Open locally by double-clicking `index.html`, or
- Run a local server for best behavior:

```bash
python3 -m http.server 8000
```

Then open `http://localhost:8000`.

## Notes
This version is intentionally general-purpose (not tied to a single company/interview).
